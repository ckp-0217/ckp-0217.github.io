Geth获取交易信息

2023-01-15 8 min read # 交易解析

geth

Geth是Go Ethereum 以太坊客户端
Geth提供了JSON-RPC API, 可以用来与它进行通信, 例如，使用 web3.js 库来访问以太坊网络, web3j库访问以太坊网络，或者使用第三方应用来与以太坊网络进行交互。

查询交易API

JSON-RPC API
https://ethereum.org/zh/developers/docs/apis/json-rpc/#json-rpc-methods
这里列几个常用的

blockHash：DATA，32 字节 - 此交易所在区块的哈希。 当它是待处理区块时，则为 null。
blockNumber：QUANTITY - 此交易所在的区块号。 当它是待处理区块时，则为 null。
from：DATA，20 字节 - 发送者的地址。
gas：QUANTITY - 发送者提供的燃料。
gasPrice：QUANTITY - 发送者提供的燃料价格，以 Wei 为单位。
hash：DATA，32 字节 - 交易的哈希。
input：DATA - 与交易一起发送的数据。
nonce：QUANTITY - 发送者在此交易之前进行的交易数量。
to：DATA，20 字节 - 接收者的地址。 当它是合约创建交易时，则为 null。
transactionIndex：QUANTITY - 表示区块中的交易索引位置的整数。 当它是待处理区块时，则为 null。
value：QUANTITY - 传输的价值，以 Wei 为单位。
v：QUANTITY - ECDSA 恢复 ID
r：QUANTITY - ECDSA 签名 r
s：QUANTITY - ECDSA 签名 s

debug_traceTransaction

callTracer

获取全部的调用信息

let trace = provider.send("debug_traceTransaction", [hash,{tracer: 'callTracer',tracerConfig: { onlyTopCall: false,withLog: true }}]);

tracer:跟踪器 这里使用的是call跟踪器
tracerConfig:
{
    onlyTopCall： true 指示跟踪器仅处理主（顶级）调用，而不处理任何子调用。
    withLog： true 指示跟踪器还收集每次调用期间发出的日志。
}
