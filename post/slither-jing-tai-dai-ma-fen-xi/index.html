slither-静态代码分析

2022-08-09 2 min read # 安全 # 审计

What is Slither

Slither is a Solidity static analysis framework written in Python 3.

How to use Slither

By VSCode
在vscode 使用slither对合约代码进行静态分析

  1. 安装python3
  2. 安装VScode C++ 组件
  3. 安装pip3 install slither-analyzer
  4. 安装VS 插件Slither
  5. 点击分析 获取分析结果

Detector Filters

过滤选项
如果对于一些风险和检测是需要跳过的，可以选择检测过滤
具体的过滤选项可以查阅另一个博文
建议全部打开

Slither Analysis

进行分析后将会产生一个分析报告

Report analysis

得到这个报告后我们要对这些检测问题进行分析和修改，有可能是业务上的决定，也有可能是真的漏洞

先将这个问题打印

⸻⸻ Detailed Result Output ⸻⸻
Type: Missing Zero Address Validation (missing-zero-check)
Impact: Low
Confidence: Medium
Finding:
❌ ManagementTest.setFeeAddress(address)._platformFeeAddress (contracts/test/ManagementTest.sol:239) lacks a zero-check on :
	• platformFeeAddress = _platformFeeAddress (contracts/test/ManagementTest.sol#240)


Recommendation: Check that the address is not zero.

Reference: https://github.com/crytic/slither/wiki/Detector-Documentation#missing-zero-address-validation

缺少零地址检测，修改只需要加一句require(!=0)就可以

再看不需要修改的例子

⸻⸻ Detailed Result Output ⸻⸻
Type: Low level calls (low-level-calls)
Impact: Informational
Confidence: High
Finding:
❌ Low level call in TransferHelper.safeTransfer(address,address,uint256) (contracts/libraries/TransferHelper.sol:8-19):
	• (success,data) = token.call(abi.encodeWithSelector(0xa9059cbb,to,value)) (contracts/libraries/TransferHelper.sol#14)


Recommendation: Avoid low-level calls. Check the call success. If the call is meant for a contract, check for code existence.

Reference: https://github.com/crytic/slither/wiki/Detector-Documentation#low-level-calls
⸻⸻⸻⸻⸻⸻⸻⸻

这里是采用了safetransfer库函数 是不需要修改的 如果不想再出现可以选择过滤
