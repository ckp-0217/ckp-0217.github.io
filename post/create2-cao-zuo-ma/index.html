Create2 操作码

2022-05-17 3 min read

Create2

君士坦丁堡硬升级中引入了一个新操作码 CREATE2 ，它使用新的方式来计算常见的合约地址，让生成的合约地址更具有可控性。

区别

Create

address = hash(msg.sender,nonce)

因为nonce是变化的所以每一次都是一样的。

Create2

address =hash("0xf",msg.sender,salt,hash(creationCode+constructorArgs))

使用

因为使用create2的时候无法传入参数

方案一:

用过调用合约方法
类似于委托代理 设置初始参数时通过一个init的函数来进行初始化
这个可以参照uniswap V2的方法

        //给bytecode变量赋值"UniswapV2Pair"合约的创建字节码
        bytes memory bytecode = type(UniswapV2Pair).creationCode;
        
        //将token0和token1打包后创建哈希
        bytes32 salt = keccak256(abi.encodePacked(token0, token1));
        
        //内联汇编
        //solium-disable-next-line
        assembly {
            //通过create2方法布署合约,并且加盐,返回地址到pair变量
            pair := create2(0, add(bytecode, 32), mload(bytecode), salt)
        }
        
        //调用pair地址的合约中的"initialize"方法,传入变量token0,token1
        IUniswapV2Pair(pair).initialize(token0, token1);

方案二:

通过查询某个地址获取参数 uniswapV3的方式

contract Creat2 {
    struct Arg {
        int256 a;
        int256 b;
    }
    Arg public arg;

    function creat2(int256 a, int256 b) public returns (address pool) {
        arg = Arg(a, b);//将参数写入 让构建方获取
        pool = address(new impl{salt: keccak256(abi.encode(a, b))}());//根据参数来设置salt
    }
    delete arg;
}

contract impl {
    uint256 public a;
    uint256 public b;

    constructor() {
        (bool success, bytes memory data) = address(msg.sender).call(
            abi.encodeWithSelector(0x7435e911)
        );
        if (success) {
            (a, b) = abi.decode(data, (uint256, uint256));
        }
    }
}

优点：可以将变量设置为immutable类型 节省gas

优势

链下计算地址 不需要链上存储

可以计算出对应地址 不论合约是否部署

                )
            )
        );
